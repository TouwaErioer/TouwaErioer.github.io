动态数组

  • 特点

    由于底层由数组data[]实现，所以是一种静态线性结构，之所以叫动态数组是可以根据情况执行扩容操作

  • 时间复杂度分析

    增加：addFrist、addLast、add均为O(n)，每次add操作都会执行for操作把元素往后移动，addLast本为O(1)但有几率执行扩容操作，所以均摊复杂度为O(n)

    删除：removeFirst、removeLast(均摊)、remove均为O(n)

    修改：set(int index)为O(1)，知道索引无需执行for操作

    查询：get()为O(1)，contains()为O(n)，int find(E e)为O(n)，知道索引为O(1)，不知道索引为O(n)

    Github

  • 思路分析

  1. 整形变量size的设置：size的作用是表示当前已装元素的数量，指向最后一个元素下一个位置，内容为null，这样设计可以巧妙的得出已装元素的数量（最后一个元素的索引+1），也同时让addLast()和removeLast()的时间复杂度为O(1)

  2. add(intindex,E e)操作：主要思路是把需插入元素data[index]的后面所有元素都向后移动一位（注意：一定要先从最后一个元素开始移动，不然其他元素会被覆盖掉），再把需插入元素e赋值给data[index]替换掉原来的元素，最后维护size的指向(size++)

  3. remove(int index)操作：主要思路是把待删除元素后面的元素依次向前移动一位将其覆盖掉（注意：从待删除元素的后面的元素data[index+1]开始依次的向前移动一位，不然其他元素会被覆盖掉），再维护size的指向(size—)，因为size移动到原来最后一个元素的位置，data[size]不等于null，所以要手动把data[size]设为null，Java会自动回收

  4. resize(int newCapacity)扩容操作：主要思路是创建一个新数组newArray[]，再把老数组data[]遍历到新数组中去

栈

  • 特点

    只允许在有序的线性数据集合的一端（称为堆栈顶端，英语：top）进行加入数据（英语：push）和移除数据（英语：pop）的运算。因而按照后进先出（LIFO, Last In First Out）的原理运作。
    堆栈常用一维数组或链表来实现，本文通过自定义Stack接口实现ArrayStack和LinkedListStack

  • 时间复杂度分析

    • ArrayStack

      pull(E e)：O(1)，均摊O(n)

      pop()：O(1)，均摊O(n)

      peek()：O(1)

    • LinkedListStack

      pull(E e)：O(1)

      pop()：O(1)

      peek()：O(1)

  • 代码实现（Java）

    Stack接口：Github

    ArrayStack：Github

    LinkedStack：Github

  • 思路分析

    由于后进先出LIFO(last in, first out)这种特性，对栈的操作都是对首（First）或尾（Last）进行操作，为了更好的性能，底层实现应选择O(1)对应的操作，比如：
    底层是ArrayStack实现，对于pull、pop、peek的实现应选择O(1)级别的addLast、removeLast、getLast；
    如果底层是LinkedStack实现，对于pull、pop的实现应选择O(1)级别的addFirst、removeFirst、getFirst；

队列

  • 特点

    是先进先出（FIFO, First-In-First-Out）的线性表。在具体应用中通常用链表或者数组来实现。队列只允许在后端（称为tail）进行插入操作，在前端（称为front）进行删除操作。

    队列的操作方式和堆栈类似，唯一的区别在于队列只允许新数据在后端进行添加。

  • 时间复杂度

    • ArrayQueue

      enQueue(E e)：O(1)，均摊O(n)—>Array(addLast)

      deQueue()：O(n)，—>Array(removeFirst)

      getFront()：O(1)，—>Array(getFirst)

    • LoopQueue

      enQueue(E e)：O(1)

      deQueue()：O(1)

      getFront()：O(1)

  • 代码实现（Java）

    ArrayQueue：Github

    LoopQueue：GIthub

  • 思路分析（LoopQueue）

  1. front的设置：front的作用是标识队首元素，让出队操作变为O(1)，每一次执行出队操作front指针向后移动一位，和ArrayQueue不同的是，每一次出队，出队元素后面的元素不会依次的向前移动一位，而是将出队元素的位置留空等于null

  2. tail的设置：tail的作用是标识下一个将要添加的元素的位置，让入队操作变为O(1)，每一次执行入队操作tail指针向后移动一位，当对队列为空的时候，front和tail都指向0

  3. 循环队列指针指向：每次出队或入队都会执行front+1或tail+1来维护指针的指向，但在循环队列里会有front前面有空位的情况，所以要遵循公式：(front+1)%data.length

    对于公式`(tail+1)%data.length`的代入参数验证：

front=0,tail=9,data.length=10->(9+1)%10=0

front=1,tail=0,data.length=10->(0+1)%10=1

front=2,tail=1,data.length=10->(1+1)%10=2

  4. 队列为满的条件：由以上公式得知(tail+1)%data.length == front的时候队列满了

  • 不同实现对比

    对ArrayQueue和LoopQueue分别执行100000次入队、出队操作，比较两个Queue所耗费的时间

链表

  • 特点

    链表（Linked list）是一种常见的基础数据结构，是一种线性表，但是并不会按线性的顺序存储数据，而是在每一个节点里存到下一个节点的指针)(Pointer)。由于不必须按顺序存储，链表在插入的时候可以达到O(1)的复杂度，比另一种线性表顺序表快得多，但是查找一个节点或者访问特定编号的节点则需要O(n)的时间，而顺序表相应的时间复杂度分别是O(logn)和O(1)。

    在计算机科学中，链表作为一种基础的数据结构可以用来生成其它类型的数据结构。链表通常由一连串节点组成，每个节点包含任意的实例数据（data fields）和一或两个用来指向上一个/或下一个节点的位置的链接（”links”）。链表最明显的好处就是，常规数组排列关联项目的方式可能不同于这些数据项目在记忆体或磁盘上顺序，数据的访问往往要在不同的排列顺序中转换。而链表是一种自我指示数据类型，因为它包含指向另一个相同类型的数据的指针（链接）。链表允许插入和移除表上任意位置上的节点，但是不允许随机存取。链表有很多种不同的类型：单向链表，双向链表以及循环链表。

    1. 单向链表

      链表中最简单的一种是单向链表，它包含两个域，一个信息域和一个指针域。这个链接指向列表中的下一个节点，而最后一个节点则指向一个空值。

      一个单向链表的节点被分成两个部分。第一个部分保存或者显示关于节点的信息，第二个部分存储下一个节点的地址。单向链表只可向一个方向遍历。

      相对于下面的双向链表，这种普通的，每个节点只有一个指针的链表也叫单向链表，或者单链表，通常用在每次都只会按顺序遍历这个链表的时候（例如图的邻接表，通常都是按固定顺序访问的）。

  • 时间复杂度

    add()操作：add(int index,E e)为O(n)，addFirst(E e)为O(1)，addLast(E e)为O(n)

    remove()操作：remove(int index)为O(n)，removeFirst()为O(1)，removeLast()为O(n)

    set()操作：set(int index,E e)为O(n)，setFirst(E e)为O(1)，setLast(E e)为O(n)

    get()操作：get(int index)为O(n)，getFirst()为O(1)，getLast())为O(n)

递归

  • 特性

    将原来的问题转换成更小的问题，最终转换成最简单的问题

    递归函数的调用和普通函数的调用一样，将调用的函数压入系统栈，等到调用的函数返回“值”的时候才执行下面的函数，无非是从A调B变成了A调A，其调用的本质是一样的

    递归的使用优点在于让程序的逻辑更清晰；缺点在于函数的调用会导致系统栈使用过多
